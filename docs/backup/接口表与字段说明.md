## 一、认证与用户

### 1.1 登录

| 项目 | 说明 |
|------|------|
| **接口名称** | 用户登录 |
| **方法** | POST |
| **路径** | `/api/auth/login` |
| **请求体** | `{ "username": string, "password": string, "remember?: boolean }` |
| **响应** | `{ "token": string, "user": UserInfo }` |
| **UserInfo** | `{ "username": string, "name": string, "role": Role, "contact?: string }` |
| **Role** | 枚举：`管理员` \| `技术标制作人` \| `商务制作人` \| `机动人员` \| `初审人员` \| `复审人员` |

### 1.2 登出

| 项目 | 说明 |
|------|------|
| **方法** | POST |
| **路径** | `/api/auth/logout` |
| **请求头** | `Authorization: Bearer <token>` |
| **响应** | `{ "success": boolean }` |

### 1.3 编辑资料（当前用户）

| 项目 | 说明 |
|------|------|
| **接口名称** | 更新个人资料 |
| **方法** | PUT |
| **路径** | `/api/user/profile` |
| **请求头** | `Authorization: Bearer <token>` |
| **请求体** | `{ "name"?: string, "contact"?: string, "newPassword"?: string }`（不修改的字段可不传） |
| **响应** | `{ "success": boolean, "user"?: UserInfo }` |

### 1.4 获取当前用户信息

| 项目 | 说明 |
|------|------|
| **方法** | GET |
| **路径** | `/api/user/me` |
| **请求头** | `Authorization: Bearer <token>` |
| **响应** | `{ "user": UserInfo }` |

## 四、标书解析

### 4.1 上传招标文件

| 项目 | 说明 |
|------|------|
| **接口名称** | 上传招标文件 |
| **方法** | POST |
| **路径** | `/api/tender/upload` |
| **Content-Type** | multipart/form-data |
| **请求体** | `file: File`（PDF/Word/TXT）；`projectId: string`（绑定项目） |
| **响应** | `{ "success": boolean, "fileId"?: string, "fileName"?: string, "size"?: string }` |

### 4.2 发起解析

| 项目 | 说明 |
|------|------|
| **接口名称** | 发起招标文件解析 |
| **方法** | POST |
| **路径** | `/api/tender/parse` |
| **请求体** | `{ "projectId": string, "fileId"?: string }` |
| **响应** | `{ "success": boolean, "taskId"?: string }`（异步时返回任务 ID） |

### 4.3 解析进度/结果（轮询或 SSE）

| 项目 | 说明 |
|------|------|
| **方法** | GET |
| **路径** | `/api/tender/parse/result` 或 `/api/tender/parse/status/:taskId` |
| **查询参数** | `projectId: string` 或 `taskId: string` |
| **响应** | `{ "status": "pending" \| "running" \| "done", "progress"?: number, "result"?: TenderParseResult }` |
| **TenderParseResult** | 见下 4.4 |

### 4.4 解析结果结构（读标结果）

| 字段/分组 | 说明 |
|-----------|------|
| **基础信息** | 招标人、代理、项目编号、预算、截止时间等 |
| **资格要求** | 资格条件列表、证明材料要求 |
| **评审要求** | 评分项、分值、评审标准 |
| **投标文件要求** | 投标文件组成、格式要求 |
| **废标/无效条款** | 无效标与废标项列表 |
| **应提交文件** | 应标需提交文件清单 |
| **招标文件审查** | 关键条款、风险提示等 |

（具体子字段可按前端已实现的「标签页 + 固定项」结构拆为接口字段。）

### 4.5 生成应标大纲

| 项目 | 说明 |
|------|------|
| **接口名称** | 生成应标大纲 |
| **方法** | POST |
| **路径** | `/api/tender/outline/generate` |
| **请求体** | `{ "projectId": string }`（可带解析结果 ID 或招标要求摘要） |
| **响应** | `{ "success": boolean, "outline"?: OutlineGroup[] }` |
| **OutlineGroup** | `{ "group": string, "sections": { "name": string, "detail"?: string }[] }` |

### 4.6 获取/更新项目应标大纲

| 项目 | 说明 |
|------|------|
| **获取** | GET `/api/projects/:projectId/outline` → `{ "tenderOutline"?: string, "techOutlineSections"?: OutlineGroup[], "bizOutlineSections"?: OutlineGroup[] }` |
| **更新** | PUT `/api/projects/:projectId/outline` 请求体同响应结构（支持大纲微调后同步到编制） |

## 十一、通用约定

| 项目 | 说明 |
|------|------|
| **Base URL** | 建议 `/api` 或按环境配置 |
| **认证** | 除登录/注册外，请求头带 `Authorization: Bearer <token>` |
| **错误响应** | `{ "code": string, "message": string }`，HTTP 状态码 4xx/5xx |
| **分页** | 统一 `page`（从 1）、`pageSize`（默认 20）；响应 `list` + `total` |
| **时间格式** | ISO 8601 或 `YYYY-MM-DD HH:mm:ss` 统一即可 |
