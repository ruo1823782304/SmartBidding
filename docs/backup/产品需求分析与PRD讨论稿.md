## 三、流程优化建议（端到端）

建议将整体流程收敛为「**以项目为主线、以大纲为纽带**」的闭环：

```
1. 资料准备（可提前做）
   ├── 历史标书入库 → AI 归类（相似处罗列）→ 同步到资质/技术方案/业绩/简历等
   └── 上传资质项、人员简历，供写标书时引用

2. 收到新招标
   └── 上传招标文件 → 解析（资格/评审/废标等）→ 【新增】生成应标大纲（标书章节结构）

3. 标书编制（与 2 绑定到同一「项目」）
   ├── 章节导航 = 本项目「应标大纲」（来自步骤 2）
   ├── 每节：复用历史/资质/简历 或 AI 生成，支持混合编辑
   ├── 任务分配：按章节分配给技术标/商务标负责人
   └── 协作审批（现有）

4. 输出与归档
   ├── 按设定标题与格式导出（Word/PDF，公司模板）
   └── 项目归档 → 标书 + 复盘 + 对手分析 进入数据中心，可检索、可复用于后续归类与推荐
```

**关键点**：

- **项目**作为一级实体：一个项目 = 一次招标 + 一份应标大纲 + 一套章节 + 任务与权限 + 归档结果。
- **大纲**来自「招标解析 + 生成应标大纲」，而不是全局固定模板；标书编制页的章节导航 = 该项目大纲。
- **资料管理**既支持「按类上传」，也支持「先上传标书 → AI 归类」，两路内容在写标书时统一可被推荐与复用。

## 四、可补充的功能点（清单，便于写 PRD）

以下按模块罗列，便于您勾选优先级并写入 PRD。

### 4.1 资料管理

- **历史标书批量上传与智能归类**：上传多份标书 → AI 分析共性维度（资质、技术方案、业绩、人员等）→ 归类罗列，并支持同步到现有分类。
- **资质/简历与章节的关联**：维护「该资质可用于哪些标书章节」或标签，写标书时按章节推荐。
- **按章节推荐素材**：在标书编制某节时，自动推荐「可用的资质 / 简历 / 历史片段」。

### 4.2 招标处理

- **生成应标大纲**：解析完成后，根据招标要求 + 公司习惯结构，生成「标书章节大纲」（一级、二级标题），并写入当前项目。
- **大纲编辑**：允许用户在生成后微调大纲（增删改章节），再同步到标书编制。

### 4.3 标书编制

- **章节导航 = 项目大纲**：左侧章节来自「当前项目的应标大纲」，而非写死结构。
- **每节双入口**：「从历史/资料复用」与「AI 生成」并存，并可混合编辑；当前节可自动推荐可复用内容。
- **导出**：按公司模板与设定标题/格式导出 Word（及 PDF），含目录、编号、封面等。

### 4.4 项目协作与权限

- **章节级任务分配**：管理员将具体章节分配给技术标/商务标负责人；编辑权限与「负责章节」绑定。
- **我的任务**：列表中展示「我负责的章节」，点击进入对应节编辑。

### 4.5 数据中心

- **项目归档**：项目结束后，将最终标书、招标、复盘、对手分析打包归档，可检索；归档标书可参与「历史标书归类」与推荐。
- **复盘结构化**：固定字段（中标/落标、原因、改进项、对手、得分拆解等），支持统计与知识库。
- **竞争对手档案**：按对手建档，多项目关联，支持在写标书或复盘时引用。

## 五、PRD 可采用的文档结构建议

1. **产品概述**：目标用户、核心价值、与现有系统的关系。
2. **角色与权限**：现有角色表 + 章节级分配与「我的任务」。
3. **功能模块**：
  - 资料管理（含历史标书归类、资质/简历、推荐逻辑）  
  - 招标处理（解析 + 应标大纲生成与编辑）  
  - 标书编制（大纲驱动导航、复用/AI、导出）  
  - 项目协作（任务分配、审批、权限）  
  - 数据中心（归档、复盘、对手分析）
4. **核心流程**：上述端到端流程图 + 关键页面/状态说明。
5. **数据与实体**：项目、大纲、章节、任务、归档、对手档案等。
6. **非功能需求**：性能、安全、审计日志（谁在何时改了哪一节）等。
7. **迭代建议**：V1 先打通「招标 → 大纲 → 章节导航」与「导出」；V2 再做历史标书归类与章节级分配等。
