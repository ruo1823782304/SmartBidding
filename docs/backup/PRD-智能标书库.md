# 智能标书库 · 产品需求规格说明（PRD）

**版本**：v1.0  
**更新日期**：2026-03-10  
**适用规模**：使用人数不超过 20 人

---

## 一、产品概述

### 1.1 产品定位

智能标书库是一套面向中小型投标团队（≤20 人）的标书全生命周期管理系统，覆盖**招标解析 → 标书编制 → 协作审批 → 归档复盘**，并通过企业库沉淀资质、业绩、人员简历与历史标书，支持复用与 AI 辅助生成，最终按设定格式输出标书。

### 1.2 目标用户

- 投标负责人 / 项目经理：统筹项目、分配任务、审查进度  
- 技术标 / 商务标制作人：编写对应章节，复用资料或使用 AI 生成  
- 初审人 / 复审人：对标书进行合规与质量审查  
- 机动人员：配合上传资料、协助编制、查看任务  
- 管理员：人员与权限、机构关系、系统设置

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 招标即大纲 | 上传招标文件后 AI 解析并生成应标大纲，标书编制章节导航与大纲一致，减少手工对齐 |
| 资料可复用 | 企业库集中管理资质、业绩、技术方案、人员简历；历史标书可入库并 AI 归类，写新标书时按章节推荐 |
| 编制可协作 | 按章节分配负责人，权限与角色绑定，支持初审/复审与驳回 |
| 输出可规范 | 按公司模板与设定标题/格式导出 Word 或 PDF |
| 数据可复盘 | 项目归档、投标复盘、竞争对手档案，支撑后续投标决策 |

## 二、系统架构与导航

### 2.1 导航结构（与设计稿一致）

系统采用左侧主导航 + 顶栏（首页/搜索/配置等），主导航分为三块：

| 区域 | 菜单项 | 说明 |
|------|--------|------|
| **MENU** | 标书解析 | 上传招标文件、AI 解析、生成应标大纲，入口与核心读标结果展示 |
| | 标书编制 | 按项目大纲编写标书，章节导航 = 本项目应标大纲，支持复用/AI 生成与导出 |
| | 企业库 | 资质、业绩、技术方案、标书归档、中标案例、人员简历；含历史标书入库与智能归类 |
| | 项目协作 | 项目看板、任务分配、流程审批、初审/复审 |
| | 数据中心 | 看板、知识库、投标复盘、竞争对手分析、项目归档 |
| **ACCOUNT** | 个人中心 | 当前用户信息、我的任务、我负责的章节、待办审批 |
| | 编辑资料 | 个人资料编辑（姓名、联系方式等） |
| **OTHER** | 设置 | 人员配置（上下级、用户与角色）、模型/API 配置、系统参数（仅管理员可见） |
| | 帮助 | 使用说明、常见问题、联系支持 |

说明：「标书解析」对应原「招标处理」能力，命名与设计稿统一；「企业库」对应原「资料管理」。

### 2.2 机构与人员规模

- **机构**：不强制多级部门，以「上下级关系」体现汇报与审批线；系统用户总数 ≤ 20。
- **人员配置页**：置于「设置」下，仅**管理员**可进入，用于维护组织关系与权限。

## 三、角色与权限

### 3.1 角色定义

| 角色 | 说明 | 典型职责 |
|------|------|----------|
| 管理员 | 系统与组织管理 | 人员配置（上下级、用户与角色）、任务分配、审查进度、系统设置、全模块可见 |
| 技术标制作人 | 技术标编写 | 技术标相关章节的编制与提交，查看/处理被分配的任务 |
| 商务标制作人 | 商务标编写 | 商务标相关章节的编制与提交，查看/处理被分配的任务 |
| 机动人员 | 灵活支持 | 配合上传资料、协助编制、查看自身任务，不单独负责固定章节 |
| 初审人 | 一审 | 对标书进行初步审查，可驳回并填写意见 |
| 复审人 | 终审 | 对标书进行最终复审，可驳回并填写意见 |

同一用户仅绑定一个角色；若需兼岗（如既做技术标又做初审），由管理员通过「任务分配」赋予具体项目的章节或审查任务，不重复建角色。

### 3.2 页面与功能权限（按角色）

| 菜单/能力 | 管理员 | 技术标制作人 | 商务标制作人 | 机动人员 | 初审人 | 复审人 |
|-----------|--------|--------------|--------------|----------|--------|--------|
| 标书解析 | ✓ | ✓ | ✓ | ✓ | — | — |
| 标书编制 | ✓ | ✓ | ✓ | ✓ | — | — |
| 企业库 | ✓ | ✓ | ✓ | ✓ | — | — |
| 项目协作 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 数据中心 | ✓ | ✓ | ✓ | ✓ | — | — |
| 个人中心 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 编辑资料 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 设置（人员配置等） | ✓ | — | — | — | — | — |
| 查看全部待办/分配任务 | ✓ | — | — | — | — | — |
| 分配章节任务 | ✓ | — | — | — | — | — |
| 初审/复审操作 | ✓ | — | — | — | 初审 | 复审 |
| 合规与质量审查页 | ✓ | — | — | — | ✓ | ✓ |

说明：初审人/复审人侧重「项目协作」中的审查与合规相关能力，可不开放标书编制与数据中心编辑，仅查看与审批相关部分。

### 3.3 章节级编辑权限

- 标书编制页中，某章节是否可编辑 = **角色是否有编制权限** ∩ **是否被分配该章节**（管理员不受分配限制）。
- 技术标制作人/商务标制作人：仅能编辑被分配给自己的章节；机动人员可按分配参与指定章节编辑。
- 未分配章节时，仅管理员可编辑全部章节（或由管理员先完成分配）。

## 四、人员配置（设置 → 人员配置）

### 4.1 功能范围

- 仅**管理员**可进入。
- 入口：**设置** → **人员配置**。
- 实现：用户列表、新增/编辑/禁用用户、分配角色、维护上下级关系。

### 4.2 用户管理

| 字段/能力 | 说明 |
|-----------|------|
| 用户名 | 登录账号，唯一 |
| 姓名 | 展示用真实姓名 |
| 密码 | 初始密码由管理员设置，支持「首次登录修改」与「编辑资料」中修改 |
| 角色 | 六选一：管理员、技术标制作人、商务标制作人、机动人员、初审人、复审人 |
| 状态 | 启用 / 禁用（禁用后不可登录，不参与任务分配） |
| 操作 | 编辑、重置密码、禁用/启用 |

约束：系统用户总数 ≤ 20；至少保留 1 名管理员。

### 4.3 上下级关系（机构维度）

| 能力 | 说明 |
|------|------|
| 配置方式 | 为每个用户指定「上级」；可选「无上级」（如最高负责人）。多级关系通过「上级」逐级形成树状结构，不要求预先建部门节点。 |
| 展示 | 以树形或列表形式展示「谁汇报给谁」，便于管理员核对机构关系。 |
| 用途 | 用于审批流、待办汇总（如「下属待办」）、统计与后续扩展；当前 PRD 不强制审批必须走上下级，但数据结构支持。 |

可选扩展：支持「审批人」单独指定（如某项目的初审人/复审人由管理员指定，而不完全按上下级），与「任务分配」一致。

### 4.4 与权限的配合

- 权限以**角色**为准（见 3.2、3.3）。
- 上下级不改变角色权限，仅用于组织关系与可选审批逻辑。

## 五、核心流程与功能说明

### 5.1 端到端流程概览

```
资料准备（可提前） → 收到新招标 → 标书解析 → 标书编制 → 协作审批 → 导出与归档
     ↓                    ↓            ↓            ↓            ↓
 企业库维护          上传招标文件   生成应标大纲  按大纲编写   初审/复审
 历史标书归类          AI 解析      同步到编制    任务分配     驳回/通过
 资质/简历上传        读标结果      章节=大纲     复用/AI生成  归档+复盘
```

### 5.2 标书解析

| 需求点 | 说明 |
|--------|------|
| 上传招标文件 | 支持 PDF、Word、TXT；上传后进入解析流程。 |
| AI 解析 | 解析出：资格要求、评审要求、投标文件要求、废标/无效条款、应提交文件、关键条款等，结构化展示（标签页或分组）。 |
| 生成应标大纲 | 解析完成后，提供「生成应标大纲」：根据招标要求 + 公司习惯结构（资信/商务/技术/其他），生成标书章节大纲（一、二、三级标题），写入当前项目。 |
| 大纲编辑 | 支持在解析页或标书编制页对大纲做微调（增删改章节），调整后同步到「标书编制」章节导航。 |
| 与项目绑定 | 每次解析绑定到「当前选中项目」；若无项目则提示先建项目或选择项目。 |

### 5.3 标书编制

| 需求点 | 说明 |
|--------|------|
| 章节导航 = 项目大纲 | 左侧章节导航来源于当前项目的「应标大纲」；若尚未生成大纲，则使用默认模板（资信/商务/技术/其他 + 常用小节）。 |
| 每节编写方式 | 每节提供「从历史/资料复用」与「AI 生成」两种方式，可混合使用：先复用再编辑，或先 AI 生成再改。 |
| 按章节推荐素材 | 根据当前章节类型，自动推荐企业库中可用的资质、业绩、技术方案、人员简历及历史标书片段，供一键插入或参考。 |
| 编辑与保存 | 支持富文本、表格、图片；实时或定时保存；可选「历史版本」查看与恢复。 |
| 任务与权限 | 仅能编辑「已分配给自己的章节」或管理员；个人中心可看到「我负责的章节」并跳转。 |
| 完成与提交 | 支持按节「标记完成」；全部章节完成后可「提交审批」或「生成标书」。 |
| 导出 | 支持按公司模板与设定标题/格式导出为 Word 或 PDF（目录、编号、封面等可配置）。 |

### 5.4 企业库

| 分类 | 能力说明 |
|------|----------|
| 企业资质 | 上传公司简介、资质清单、证照等；支持「整体上传」多份后由 AI 解析并归类到对应子项。 |
| 项目业绩 | 业绩表、合同关键页等；支持筛选与关联到标书章节。 |
| 技术方案 | 技术方案文档上传、分类管理；写技术标时可按章节推荐。 |
| 标书归档 | 历史标书归档存储；可与「历史标书入库与智能归类」打通。 |
| 中标案例 | 中标项目案例库，写标书时引用。 |
| 人员简历 | 上传简历、AI 提取关键信息、模板填充；写标书时按「项目团队」等章节推荐。 |
| 历史标书入库与智能归类 | 批量上传多份已编写标书 → AI 分析共性维度（资质、技术方案、业绩、人员等）→ 归类罗列，结果可同步到上述分类，供编制时推荐与复用。 |

### 5.5 项目协作

| 需求点 | 说明 |
|--------|------|
| 项目看板 | 项目列表/看板（待开始、进行中、审查中、已完成），支持按类型筛选、拖拽状态。 |
| 任务分配 | 管理员（或具备权限者）为项目分配「章节负责人」：选择用户 + 章节（可多选），技术标/商务标制作人、机动人员可被分配。 |
| 待办与进度 | 各角色在「个人中心」或「项目协作」查看自己的任务、负责的章节及完成进度。 |
| 初审/复审 | 初审人、复审人对待提交标书进行审查，可「通过」或「驳回」并填写意见；驳回后任务回到编制人。 |
| 流程状态 | 标书状态：编制中 → 待初审 → 初审通过/驳回 → 待复审 → 复审通过/驳回 → 可归档。 |

### 5.6 数据中心

| 模块 | 说明 |
|------|------|
| 看板 | 项目统计、中标率趋势、当前进行中项目概览等。 |
| 知识库管理 | 复盘与案例沉淀为知识条目，可分类、检索、导出。 |
| 投标复盘 | 按项目做复盘：中标/落标、原因、改进项、得分拆解、对手情况等结构化字段，支持统计与检索。 |
| 竞争对手分析 | 按对手建档：名称、价格、优劣势、常用策略等；可关联到多个项目，写标书或复盘时引用。 |
| 项目归档 | 项目结束后，将最终标书、招标文件、复盘、对手分析打包归档，形成可检索项目库；归档标书可参与「历史标书归类」与推荐。 |

## 六、数据实体（核心）

| 实体 | 主要字段/关系 |
|------|----------------|
| 用户 | 用户名、姓名、密码、角色、上级、状态、创建/更新时间 |
| 项目 | 名称、状态、截止时间、关联招标文件、应标大纲、进度、创建人/时间 |
| 应标大纲 | 项目 ID、层级结构（一/二/三级标题）、顺序、可选与默认模板关联 |
| 章节任务 | 项目 ID、章节 ID、负责人用户 ID、状态、完成时间 |
| 标书内容 | 项目 ID、章节 ID、内容（富文本/版本）、最后编辑人/时间 |
| 审批记录 | 项目 ID、审批人、初审/复审、通过/驳回、意见、时间 |
| 企业库资料 | 分类、标题、文件或文本、标签、上传人/时间；历史标书归类结果可写回或关联 |
| 竞争对手 | 名称、价格、优劣势、策略、关联项目 ID 列表 |
| 复盘记录 | 项目 ID、中标/落标、原因、改进项、得分拆解、对手 ID 等 |

## 七、非功能需求

| 类型 | 要求 |
|------|------|
| 规模 | 用户数 ≤ 20；支持项目数、标书数量按实际使用扩展。 |
| 安全 | 登录认证、密码加密存储；敏感操作（人员配置、删除、导出）可记审计日志。 |
| 审计 | 关键操作留痕：谁在何时修改了哪一章节、谁执行了审批与驳回等，便于追溯。 |
| 性能 | 列表与编辑页首屏可接受 2s 内响应；大文档解析与 AI 生成可异步并提示进度。 |
| 浏览器 | 支持现代浏览器（Chrome、Edge、Safari 等），建议分辨率 1280×720 及以上。 |
