# 智能标书库 · 技术选型

**适用场景**：≤20 人团队，依据 PRD 全生命周期标书管理。  
**选型日期**：2026-03

---

## 一、前端

| 类别 | 技术 | 说明 |
|------|------|------|
| 框架 | **React 18+** | 与当前项目一致，组件化与生态成熟 |
| 语言 | **TypeScript** | 类型安全，与后端 DTO 对齐 |
| 构建 | **Vite** | 开发/构建，已在使用 |
| 样式 | **Tailwind CSS** | 工具类 + 现有 Mac OS 风变量 |
| 组件库 | **shadcn/ui** | 与当前 UI 风格一致，可定制 |
| 请求/缓存 | **TanStack Query (React Query)** | 接口请求、缓存、轮询（解析进度等） |
| 表单 | **React Hook Form + Zod** | 表单与校验（新增用户、编辑资料、任务分配等） |
| 路由 | **React Router**（可选） | 若从单页拆多路由时可引入 |

---

## 二、后端

| 类别 | 技术 | 说明 |
|------|------|------|
| 框架 | **NestJS** | Node.js + TypeScript，模块化（Auth / Project / Tender / Proposal / Collaboration / Asset / Admin），内置守卫、管道、Swagger |
| 数据库 | **PostgreSQL** | 主库：用户、项目、大纲、章节内容、任务、审批、企业库等 |
| ORM | **Prisma** | 建模、迁移、类型安全访问 |
| 缓存/队列 | **Redis** | 会话、限流、可选任务队列 |
| 对象存储 | **MinIO（开发）/ 阿里云 OSS 或 S3（生产）** | 招标文件、企业库文件、导出 Word/PDF |
| 异步任务 | **BullMQ**（基于 Redis）或 **Nest Schedule + 队列** | 招标解析、AI 生成、导出等耗时任务 |

---

## 三、AI 与文档

| 类别 | 技术 | 说明 |
|------|------|------|
| 大模型 | **OpenAI / 通义千问 / DeepSeek** | 按「设置 → 模型配置」切换，读标、生成大纲、按章节生成等 |
| PDF 解析 | **pdf-parse** 或 **pdfjs-dist** | 招标文件正文抽取 |
| Word 解析 | **mammoth** | Word 转 HTML/文本 |
| 导出 Word | **docx** 或 **docxtemplater** | 按模板生成标书 Word |
| 导出 PDF | 前端打印或服务端渲染 | 视排版要求选择 |

---

## 四、鉴权与权限

| 类别 | 技术/方案 | 说明 |
|------|------------|------|
| 认证 | **JWT + Refresh Token**（或先简化为单 JWT） | 登录后签发，请求头 `Authorization: Bearer <token>` |
| 权限 | **RBAC + 章节级 ACL** | 角色控制页面/能力；`section_assignments`（项目 + 章节 + 负责人）控制标书编制编辑权 |

---

## 五、工程与部署

| 类别 | 技术 | 说明 |
|------|------|------|
| 接口文档 | **Swagger (OpenAPI)** | NestJS 集成，与接口表对齐 |
| 容器化 | **Docker + Docker Compose** | 编排：PostgreSQL、Redis、MinIO、后端 API、前端静态 |
| 日志 | **Pino** | 结构化日志 |
| 监控 | **Prometheus + Grafana**（可选） | MVP 可暂不上 |

---

## 六、与现有项目的关系

- **前端**：当前仓库已为 **React + TypeScript + Vite + Tailwind + shadcn/ui**，仅需补充 **TanStack Query**、**React Hook Form + Zod**，并按接口表接入后端。
- **后端**：新建 NestJS 仓库或 monorepo 下 `apps/api`，按 `docs/接口表与字段说明.md` 实现接口；数据库用 Prisma 建表，与 PRD 数据实体一致。
